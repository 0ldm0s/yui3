Base is designed to be a low level class from which other attribute and event target based classes in the YUI library can be derived. It provides a standard template for creating attribute based objects across the library and a consistent init() and destroy() sequence for objects derived from Base, by chaining initialization (initializer) and destruction (destructor) methods for the class hierarcy.

Extending Base

Although Base can be instantiated, it's really designed to be a root class, which you extend when creating your own Attribute and Event.Target based classes as shown below:

Base itself augments Attribute, which in turn augments Event.Target, making Base an Attribute provider and an Event Target.

Base's contructor expects a configuration object literal, which is uses to setup user provides values for attributes (discussed below).

ATTRS

The ATTRS property is an associative array (object literal) which is used to define the default set of attributes the class adds to the instance. The instance will contain attributes defined by each class in the class hierarchy from which it is created, which each class adding the set of attributes and supporting code which it brings to the picture.

Each key in the associative array (e.g. "dragNode"), defines the name of the attribute to be added, and the corresponding value defines the object literal used to configure the attribute. See Attribute's discussion of configuration properties for more details about the structure of configuration object for each attribute.

When instantiating an instance of a class which is derived from Base, Base's init() method , will initialize the set of attributes defined by the ATTRS property for each class in the class heirarchy. This helps avoid replicating attribute initialization code in the constructor/initializer of each class and defines a specific order in which attributes are initialized - starting from the Base class first, and ending with the specific sub-class being instantiated.

Initialization and Destruction

Base implements final versions of it's init and destroy methods used to establish the init and destroy lifecycle phases for all classes derived from Base. Classes extending Base can perform operations during initialization or destruction, by defining prototype level initializer and destructor methods respectively for the class:

Base's init and destroy methods take care of invoking intializer and destructor methods for each class in the hierarchy. The implementations for each class do not need to call superclass versions of the method. This way Base can ensure that intialization and destruction occur in a consistent, manner, following the class hierarchy.

initializer()
Base will invoke the initializer method for each class in the hierarchy - starting from the Base class first and ending with the specific sub-class being instantiated, when init(cfg) is called, which by default is done in Base's constructor. The initializer method for the class is invoked after it's attributes have been initialized (as discussed above) and will receive the configuration object literal passed to the init method.
destructor()
Base will invoke the destructor method for each class in the hierarchy - starting from the specific sub-class instantiated to create the instance, and ending with the Base class (the opposite of initialization), when destroy() is called.
