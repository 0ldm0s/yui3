Base is designed to be a low level class from which other attribute and event target based classes in the YUI library can be derived. It provides a standard template for creating attribute based objects across the library and a consistent init() and destroy() sequence for objects derived from Base, by chaining initialization (initializer) and destruction (destructor) methods for the class hierarcy.

ATTRS

The ATTRS property is an associative array (object literal) which is used to define the default set of attributes the class adds to the instance. The instance will contain attributes defined by each class in the class hierarchy from which it is created, which each class adding the set of attributes and supporting code which it brings to the picture.

See Attribute's discussion of configuration properties for more details about the structure of configuration object for each attribute.

When instantiating an instance of a class which is derived from Base, Base's init() method , will initialize the set of attributes defined by the ATTRS property for each class in the class heirarchy. This helps avoid replicating attribute initialization code in the constructor/initializer of each class and defines a specific order in which attributes are initialized - starting from the Base class first, and ending with the specific sub-class being instantiated.

Initialization and Destruction

Base implements final versions of it's init and destroy methods used to establish the init and destroy lifecycle phases for all classes derived from Base. Classes extending Base can perform operations during initialization or destruction, by defining prototype level initializer and destructor methods respectively for the class:

Base's init and destroy methods take care of invoking intializer and destructor methods for each class in the hierarchy. The implementations for each class do not need to call superclass versions of the method. This way Base can ensure that intialization and destruction occur in a fixed order, following the class hierarchy.

initializer()
Base's init method will invoke the initializer method for each class in the hierarchy - starting from the Base class first and ending with the sub-class being instantiated, when it is called, which by default is in Base's constructor. The initializer method for the class is invoked after it's attributes have been initialized (as discussed above) and will receive the configuration object literal passed to the init method.
destructor()
Base's destroy method when called, will invoke the destructor method for each class in the hierarchy - starting from the sub-class instantiated to create the instance, and ending with the Base class (the opposite of initialization).

If your class does not require any code to be executed during either of these phases, these methods do not need to be defined on it's prototype.

Build and Create

In addition to the class hierarchy based support discussed above, the Base class also provides two static methods build and create, used to create custom classes or instances of classes derived from Base.

NOTE: These two methods are not currently being used by code in the 3.0.0 PR1 release, but are indended for use as we progress towards a modular widget infrastructure and may be revisited in future pre-beta releases as requirements are refined.

Build

The build method can be used to dynamically create new classes, which are derived from an existing main class and mix in additional classes to add methods, attributes, events and properties to the main class. By default, it leaves the original class and mixed in classes untouched, so that they can be used without the additional features mixed in, however it can also be configured to modify the main class if required.

Under the hood, build:

  • Creates a new 'built' class by extending the main class passed in as the first argument.
  • Augments the list of feature classes (the 2nd argument) to built class, adding their prototype methods to it.
  • Aggregates known static properties (by default ATTRS on the built class. The caller can provide other properties to aggregate also, using the 3rd configuration argument to build.

The new class constructor created by build will invoke the constructors for the main and feature classes when the new class is instantiated.
